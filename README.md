# Universal Governance Compiler (UGC)

UGC compiles one repository-local governance source into agent-specific governance files.

## V1 Scope

V1 targets:

- OpenAI Codex: `AGENTS.md`, `.codex/config.toml`, `.codex/rules/ugc.rules`
- Google Antigravity: `.agents/AGENTS.md` and `.agents/skills/*/SKILL.md`
- Claude Code: `CLAUDE.md`, `.claude/settings.json`
- Cursor: `.cursorrules`

Copilot, Windsurf, and community emitters are deferred beyond V1. Git self-defense hooks and CI enforcement are V2 work, not V1.

## What UGC Verifies

- `ugc init` creates a UGC-native `.universal-governance/` corpus and versioned corpus state.
- `ugc build --dry-run` previews generated target writes without changing files.
- `ugc build` emits deterministic V1 target files for Codex, Antigravity, Claude, and Cursor, refuses to overwrite unmanaged existing target files, applies generated artifacts transactionally with local rollback on apply failure, then writes a build manifest last.
- `ugc audit` checks source structure, generated-file drift, missing files, unexpected stale files, build manifest consistency, target capability coverage, and corpus/update state.
- `ugc packet new/hash/verify` creates hash-bound approval packet templates, prints packet SHA256, and verifies approval text before scoped execution.
- `ugc update --dry-run` previews corpus changes without writing files.

## V1 Enforcement Labels

- Claude Code is `constrained` for conservative deny rules emitted in `.claude/settings.json`; V1 does not claim comprehensive hard enforcement without a verified PreToolUse hook.
- OpenAI Codex is `constrained` for approval/sandbox defaults and project-local command rules that load only when the project `.codex/` layer is trusted. Codex secret-read protection is `advisory` in V1.
- Antigravity is `native-skill` / `instructed`, not machine-enforced.
- Cursor is `instructed` in V1. Cursor hook enforcement is not emitted because the exact local hook schema was not verified for this implementation packet.

## Audit Boundaries

UGC owns and audits exact generated paths such as `.claude/settings.json`, `.codex/config.toml`, and `.codex/rules/ugc.rules`. It does not treat normal vendor/user files under `.claude/` or user rules under `.codex/rules/*.rules` as unexpected generated artifacts.

## Restore Flow

If `ugc audit` reports drift in a UGC-owned generated artifact, restore that exact path:

```bash
ugc audit
ugc build --restore .claude/settings.json
ugc audit
```

`--restore` only accepts exact artifact paths already owned by the build manifest and currently generated by UGC. It is not a broad force mode.

## What UGC Does Not Guarantee

UGC verifies generated governance artifacts. It does not guarantee that every third-party AI agent will obey instructions identically at runtime.

UGC preflights generated build output, preserves prior managed file state, applies generated files with local rollback on apply failure, and writes the build manifest last. If rollback cannot fully restore state because the filesystem itself is unavailable or refuses the restore, the build reports that clearly and no clean manifest is written; run `ugc audit` after any interruption or incomplete rollback.

`ugc packet verify` is a local discipline aid for hash-bound approval text. It checks literal approval fields, the current packet hash, and the declared packet path, but it is not a cryptographic identity or intent-signing system.

## Basic Flow

```bash
ugc init
ugc build --dry-run
ugc build
ugc audit
ugc build --restore .claude/settings.json
ugc packet new --dry-run --task-id UGC-EXAMPLE-001 --path Plans/UGC_Example_Packet.md --source Plans/source.md --masterplan Plans/Product_Masterplan.md
ugc packet hash Plans/UGC_Example_Packet.md
ugc packet verify --packet Plans/UGC_Example_Packet.md --approval "<approval sentence>"
ugc update --dry-run
```
